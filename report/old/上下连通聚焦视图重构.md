# 上下连通（TOP_BOTTOM_CONNECTED）聚焦视图重构

## 概述

对上下连通模式（18行棋盘）的己方视角（SELF）和敌方视角（OPPONENT）进行了全面的渲染和逻辑重构。

## 修改文件

### 1. `BoardPanel.java` — 聚焦视图渲染和交互

#### 1.1 `buildFocusedRows()` — 行排列逻辑
- **旧逻辑**：写死 21 行数组，末尾有重复的 `oppR` 元素，导致楚河汉界上下各多显示一行
- **新逻辑**：19 行环绕排列，从对手将/帅所在行出发，`(oppR+k)%18` 环绕一圈回到起点
- 河界不再作为独立数组元素（`-1`），改为在两个连续显示行之间用 ±2 双线表达
- 联机模式下根据 `localControlsRed` 动态确定己方颜色：
  - SELF 视角：显示自己控制的颜色在中间
  - OPPONENT 视角：显示对方颜色在中间

#### 1.2 `drawFocused()` — 聚焦视图绘制
- 河界占用两个连续显示行（disp4-5 / disp13-14），横线用 ±2 双线（仿 `drawBoardGrid` 的 `isRiverRow`）
- 竖线在河界两行之间断开
- 河界文字调用 `drawRiverLabel()`，与全局视图完全一致

#### 1.3 宫线绘制
- 宫线固定在逻辑行 row13（红）和 row4（黑），不再跟随将/帅的实时位置移动
- 最底下的重复将/帅（数组末元素）只画上方米字（`drawLower=false`）

#### 1.4 选中高亮和移动指示器
- 新增 `sdr`（selectedDisplayRow）字段，点击时记录实际显示行
- `drawMoveIndicators` 中：
  - 黄色选中高亮：遍历 `buildFocusedRows` 所有匹配行，每个都高亮
  - 绿色可行走点：同样遍历所有匹配显示行
- 取消选中检查新增 `dr==sdr` 条件，避免点击底部副本被误判为重复点击

#### 1.5 晋升底线判断
- 上下连通模式：敌方底线 = row4（黑）/ row13（红），己方底线 = row13（红）/ row4（黑）
- 标准模式：保持不变（0 / rows-1）

### 2. `MoveValidator.java` — 兵卒过河逻辑

#### 2.1 `isValidSoldierMove()` — 修复过河判断
- **BUG**：`crossed` 判断方向完全相反（`fr>riverRow` 应为 `fr<=riverRow`）
- 导致所有兵卒开局即被视为"已过河"，可以自由左右移动
- **修复**：翻转不等号 `crossed = isRed ? fr<=riverRow : fr>riverRow`

#### 2.2 `hasCrossedRiver()` — 抽取公共方法
- 新增 `hasCrossedRiver(int row, boolean isRed)` 方法，自动根据 `tb()` 选 riverRow=8 或 4
- `isValidSoldierMove` 已改用此方法

### 3. `GameEngine.java` — 晋升底线判断

#### 3.1 `makeMove()` 中的晋升检查
- **旧逻辑**：写死 `toRow == 0` / `toRow == 9`（标准模式）
- **新逻辑**：连通模式用 row4（黑）/ row13（红），标准模式用 `board.getRows()-1`

#### 3.2 `needsPromotion()` 中的晋升检查
- 同上修改

## 架构变化

| 项目 | 修改前 | 修改后 |
|------|--------|--------|
| 聚焦行数 | 21 行（含重复行 + 河界占行） | 19 行（河界不占行） |
| 河界表达 | 数组中 `-1` 哨兵 + 独立行高 | 两个连续显示行 ±2 双线 |
| 宫线位置 | 跟随将/帅实时位置 | 固定在 row13（红）/ row4（黑） |
| 选中高亮 | `logicToDisplay` 仅返第一个匹配 | 遍历所有匹配显示行 |
| 兵卒过河 | 方向反了（始终判定已过河） | 正确判定（`hasCrossedRiver`） |
| 晋升底线 | 写死 0/9 | tb() 时用 row4/row13 |
| 联机视角 | 写死 SELF=红 | 根据 `localControlsRed` 动态确定 |
