## 二、穿透分析：MCTSAgent 真的「用」了神经网络吗？

### 2.1 Value✅ / Policy❌  — 只用了价值，没用到策略

这是**之前所有审计遗漏的最关键问题**。

看一看 [`MCTSAgent.java:129-144`](ucc-ai/src/main/java/io/github/samera2022/chinese_chess/ai/MCTSAgent.java:129) 的真实路径：

```java
// ── 评估阶段 ──
if (inferenceFunction != null) {
    float[] nnResult = inferenceFunction.apply(forkCtx, null).get();
    // nnResult[0] = value, nnResult[1...] = policy (此处只取value)
    value = nnResult.length > 0 ? nnResult[0] : 0.0;  // ← 只取了value!
    // 注意：nnResult[1...] 即 policy 被完全丢弃！
}
```

再看 [`selectBestChild()`](ucc-ai/src/main/java/io/github/samera2022/chinese_chess/ai/MCTSAgent.java:292) (UCB公式)：

```java
double exploration = EXPLORATION_CONSTANT    // 1.414
    * Math.sqrt(Math.log(parent.visitCount) / child.visitCount);
// 注意！这是标准UCB1，不是AlphaZero的PUCT！
// AlphaZero: C * P(s,a) * sqrt(N) / (1+n)
// 这里:       1.414 * sqrt(ln(N)/n)        ← priorPolicy未被使用
```

**`MCTSNode.priorPolicy`** 字段（第45行）虽然存在，但**没有任何代码去读它**。

### 2.2 这意味着什么？

| 对比项 | 标准 AlphaZero | 当前实现 |
|--------|:-------------:|:--------:|
| 神经网络 policy 指导搜索 | ✅ PUCT 公式使用 P(s,a) | ❌ 完全忽略 |
| 神经网络 value 评估局面 | ✅ | ✅ 使用 |
| 训练 policy target | ✅ 来自 MCTS 访问计数 | ✅ 来自 MCTS 访问计数 |
| 训练 value target | ✅ 终局胜负 | ✅ 终局回填 |

**影响评估**：这依然能训练，但：
- MCTS 搜索效率较低（没有策略先验引导，每次搜索都需要更多的模拟次数才能聚焦）
- 当前 `numSimulations=800` 的配置，实际效果可能接近于纯 value-based MCTS
- 这不是致命的——DeepMind 早期的作品也是先有 value network 再有 policy network

### 3.2 🥚🐔 鸡生蛋问题 — 初始模型从哪来？

[`inference_server.py:33-43`](ucc-ai/python/inference_server.py:33) 的构造函数明确要求：

```python
def __init__(self, model_path: str, device: str = "cuda"):
    checkpoint = torch.load(model_path, map_location=self.device)  # ← 必须存在
```

同时在 [`inference_server.py:163`](ucc-ai/python/inference_server.py:163) 中：

```python
parser.add_argument("--model", type=str, required=True, help="模型检查点路径")
```

**这意味着你必须先有一个 .pt 文件才能启动推理服务。** 而 .pt 文件只能通过训练得到。

**解决路径**：需要先跑一次 Python 独立训练生成初始模型，再启动整套系统。

## 四、实际操作：端到端运行步骤

### 4.1 完整启动序列

```
Step 1: 生成初始模型（只需一次）
  cd ucc-ai/python
  python train.py           ← 生成 checkpoints_test/model_final.pt

Step 2: 启动推理服务
  python inference_server.py --model checkpoints_test/model_final.pt --port 50051
  → 等待输出: "推理服务已启动，端口: 50051"

Step 3: 启动训练服务
  python train_server.py --port 50052 --checkpoint checkpoints_test/model_final.pt
  → 等待输出: "训练 gRPC 服务已启动，端口: 50052"

Step 4: 启动 Java 服务器
  cd ../..
  mvn clean compile exec:java -pl ucc-server
  → TrainingOrchestrator 开始 2000 轮课程学习
```

### 4.2 鸡生蛋问题的变通方案

也可以不用先跑 `train.py`，而是生成一个随机权重的 checkpoint：

```python
# save_random_model.py
import torch
from model import MiniResNet
from selfplay import NUM_CHANNELS, EXPANDED_ROWS, BOARD_COLS, RULE_FULL_DIM
model = MiniResNet(NUM_CHANNELS, EXPANDED_ROWS, BOARD_COLS, RULE_FULL_DIM)
torch.save({"model_state_dict": model.state_dict(), "iteration": 0}, "checkpoints/init_model.pt")
```
