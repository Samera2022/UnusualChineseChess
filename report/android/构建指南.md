# Unusual Chinese Chess Android 构建指南

---

## 1. 前置条件

| 软件 | 版本要求 | 说明 |
|------|---------|------|
| JDK | 17 (推荐) | `app/build.gradle.kts` 已设 `VERSION_17`。JDK 21 也可用 |
| Android SDK | API 34 (compileSdk) | 需安装 `platforms;android-34` 和 `build-tools` |
| Gradle | 8.2+ | 建议使用 Gradle Wrapper (自动下载) |
| Android Studio | 最新稳定版 | 推荐, 非必需 |

### 检查当前环境

```bash
# 检查 JDK
java -version

# 检查 Android SDK 路径 (如果已安装)
echo %ANDROID_HOME%
echo %ANDROID_SDK_ROOT%
```

## 2. 配置 Android SDK

### 2.1 方式一: 通过 Android Studio 安装 (推荐)

1. 下载安装 [Android Studio](https://developer.android.com/studio)
2. 启动后进入 **SDK Manager** (`Tools` → `SDK Manager`)
3. 确保已安装:
   - **Android 14.0 (API 34)** — `platforms;android-34`
   - **Android SDK Build-Tools** (最新版)
   - **Android SDK Platform-Tools** (`adb` 等)
4. 记下 SDK 路径 (通常在 `C:\Users\<用户名>\AppData\Local\Android\Sdk`)

### 2.2 方式二: 通过命令行安装 (sdkmanager)

```bash
# 下载 command-line tools
# https://developer.android.com/studio#command-line-tools-only

# 解压到 C:\Android\cmdline-tools

# 安装 API 34
sdkmanager "platforms;android-34"
sdkmanager "build-tools;34.0.0"
```

## 3. 配置项目

### 3.1 设置 SDK 路径

编辑 [`ucc-android/local.properties`](ucc-android/local.properties), 将 `/path/to/Android/sdk` 替换为你的实际 SDK 路径:

```
# Windows 示例
sdk.dir=C\:\\Users\\_S_A_M\\AppData\\Local\\Android\\Sdk

# 或使用正斜杠
sdk.dir=C:/Users/_S_A_M/AppData/Local/Android/Sdk
```

> **注意**: `local.properties` 是个人环境配置, 不要提交到 Git。

### 3.2 关于项目目录结构

Android 项目 `ucc-android/` 通过 `sourceSets` 直接引用同级目录的源码:

```
UnusualChineseChess/
├── ucc-common/src/main/java/     ← 共享模型类 (直接编译)
├── ucc-core/src/main/java/       ← 核心引擎 (直接编译)
├── ucc-api/src/main/java/        ← 序列化/网络层 (直接编译)
└── ucc-android/                  ← Android 项目根目录
    └── app/src/main/java/        ← Android UI + 适配层
```

所以**必须保持这些目录的相对位置不变**, 否则编译失败。详细见 [`app/build.gradle.kts`](ucc-android/app/build.gradle.kts) 中的 `sourceSets` 配置。

## 4. 构建方式

### 4.1 方式一: Android Studio (推荐)

1. **打开项目**: `File` → `Open` → 选择 `ucc-android/` 目录
2. **等待 Gradle 同步**: Android Studio 会自动识别配置并同步
3. **配置 SDK**: 如果提示 SDK 未找到, 在 `local.properties` 中设置 `sdk.dir`
4. **运行**: 点击绿色 ▶ 按钮, 选择设备/模拟器

> **同步完成前** Android Studio 可能会显示编译错误。这是因为 Android Studio 不直接识别 `sourceSets` 引用外部目录的方式, 需要运行 Gradle 同步后才会正确解析。**耐心等待同步完成即可。**

### 4.2 方式二: 命令行构建

```bash
# 进入 Android 项目目录
cd ucc-android

# 生成 Gradle Wrapper (如果尚不存在)
gradle wrapper --gradle-version=8.2.2

# 构建 Debug APK
./gradlew assembleDebug

# 构建 Release APK
./gradlew assembleRelease

# 安装到已连接的设备
./gradlew installDebug

# 清理构建产物
./gradlew clean
```

### 4.3 添加 Gradle Wrapper (推荐)

如果不想依赖本机安装的 Gradle, 可以生成 Gradle Wrapper:

```bash
# 在 ucc-android/ 目录下执行
gradle wrapper --gradle-version=8.2.2
```

这会生成:
- `gradlew` (Windows 批处理)
- `gradlew.bat` (Unix shell)
- `gradle/wrapper/gradle-wrapper.jar`
- `gradle/wrapper/gradle-wrapper.properties`

之后就可以直接用 `./gradlew` 构建, 无需本机安装 Gradle。

## 5. 常见问题

### 5.1 编译报错: SDK 未找到

```
Failed to find target android-34
```

**原因**: `local.properties` 中 `sdk.dir` 未配置或指向了不存在的路径。

**解决**: 正确设置 SDK 路径。

### 5.2 编译报错: ucc-common 源码找不到

**原因**: 构建时 `sourceSets` 中的 `../ucc-common/src/main/java` 路径不存在。

**解决**: 确保从 `ucc-android/` 目录往上两级能找到 `ucc-common/` 目录。不要单独移动 `ucc-android/` 文件夹。

### 5.3 编译报错: Java 版本不匹配

```
Unsupported class file major version 67
```

**原因**: JDK 版本过高/低。

**解决**: 
```bash
# 确保 JAVA_HOME 指向 JDK 17+
set JAVA_HOME=C:\Program Files\Java\jdk-17
```

### 5.4 运行时报错: 网络权限

**原因**: 局域网对局需要网络权限。已在 [`AndroidManifest.xml`](ucc-android/app/src/main/AndroidManifest.xml) 中声明:

```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_WIFI_STATE" />
<uses-permission android:name="android.permission.CHANGE_WIFI_MULTICAST_STATE" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
```

如果使用 Android 12+, 还需确保应用在前台运行时使用网络。

### 5.5 运行时报错: mDNS 发现不工作

mDNS 需要 `ACCESS_WIFI_STATE` 和 `CHANGE_WIFI_MULTICAST_STATE` 权限, 这些权限在 Android 6+ 属于普通权限(自动授予), 不需要运行时请求。

但如果设备不支持 mDNS (部分定制 ROM), 可手动输入 IP 地址作为 fallback。

## 7. 快速开始 (TL;DR)

```bash
# 1. 安装 Android Studio 和 API 34

# 2. 配置 SDK 路径
echo "sdk.dir=C:/Users/_S_A_M/AppData/Local/Android/Sdk" > ucc-android/local.properties

# 3. 用 Android Studio 打开 ucc-android/ 目录

# 4. 等待 Gradle 同步完成 → 点击 Run
```

或纯命令行:

```bash
# 1. 安装 JDK 17+
# 2. 安装 Android command-line tools 并安装 API 34

# 3. 配置 SDK 路径
echo "sdk.dir=C:/Users/_S_A_M/AppData/Local/Android/Sdk" > ucc-android/local.properties

# 4. 构建
cd ucc-android
gradle wrapper --gradle-version=8.2.2
./gradlew assembleDebug

# 5. APK 位置: app/build/outputs/apk/debug/app-debug.apk
```
