## 关键重构对比

### ChineseChessFrame 拆分

| 重构前 | 重构后 |
|--------|--------|
| 1 个类 889 行 | 4 个类，各自 ≤ 250 行 |
| 直接处理 NetworkSession.Listener 的 15 个回调方法 | 委托给 `NetGameCoordinator` |
| 内嵌 ForceMove 请求/确认/重试/超时逻辑 | 独立的 `ForceMoveHandler` |
| 混杂走子协调和状态更新 | 提取到 `GameController` |

### BoardPanel 向上穿透

| 重构前 | 重构后 |
|--------|--------|
| `getNetController()` 通过 `getWindowAncestor()` 向上穿透 | 构造函数注入 `BooleanSupplier`, `Consumer<JsonObject>` |
| 直接访问 `ChineseChessFrame.netController` | 不再感知父窗口类型 |
| 紧耦合到具体 UI 类 | 仅依赖函数式接口 |

### IO 层解耦

| 重构前 | 重构后 |
|--------|--------|
| `exportGameState(GameEngine, String)` | `exportGameState(GameStateAccessor, Board, String)` |
| `importGameState(GameEngine, String)` | `importGameState(GameStateAccessor, Board, String)` |
| 直接调用 `GameEngine` 内部方法 + 反射访问私有字段 | 通过 `GameStateAccessor` 接口 + `GameStateListener` |
| `ucc-api` 需要依赖 `ucc-core` | `ucc-api` 只需依赖 `ucc-common` |

### JSON 序列化

| 重构前 | 重构后 |
|--------|--------|
| `getSyncState()` 在 `GameEngine` 内 (~85行) | 移至 `ucc-api.GameStateSerializer` |
| `loadSyncState()` 在 `GameEngine` 内 (~55行) | 移至 `ucc-api.GameStateSerializer` |
| 引擎层依赖 Gson 内部细节 | 引擎层纯业务逻辑，序列化关注点分离 |
