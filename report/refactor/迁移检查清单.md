- [ ] 创建 `spi/ReadonlyBoard.java`（只读棋盘 → ucc-ai 预留）
- [ ] 创建 `spi/SimulationContext.java`（AI 模拟走子沙箱 → ucc-ai 预留）
- [ ] 创建 `spi/AIStrategy.java`（AI 策略接口 → ucc-ai 预留）
- [ ] 创建 `spi/AIStrategyConfig.java`（AI 参数配置值对象 → ucc-ai 预留）
- [ ] 创建 `spi/MatchSession.java`（服务端对局会话 → ucc-server 预留）
- [ ] 创建 `spi/MatchmakingService.java`（匹配/房间服务 → ucc-server 预留）
- [ ] 创建 `spi/MessageProtocol.java`（WebSocket 消息协议常量 → ucc-server 预留）

以上均为纯接口/常量类，不含实现。创建后立即 `mvn clean compile` 验证编译通过。

### 3.2 重构 IO 层解耦

- [ ] 创建 `ucc-api/.../api/io/GameStateSerializer.java`

  - 迁移 `GameEngine.loadSyncState()` 逻辑 → `deserialize(GameStateAccessor, Board, JsonObject)`

- [ ] `io/GameStateExporter.java` → `ucc-api/.../api/io/GameStateExporter.java`
  - 方法签名从 `exportGameState(GameEngine, String)` → `exportGameState(GameStateAccessor, Board, String)`
  - 更新 import
- [ ] `io/GameStateImporter.java` → `ucc-api/.../api/io/GameStateImporter.java`
  - 方法签名从 `importGameState(GameEngine, String)` → `importGameState(GameStateAccessor, Board, String)`
  - 去除反射调用 `onMoveExecuted`/`onGameStateChanged`（改用 `GameStateListener`）
  - 更新 import

### 3.3 迁移网络层

- [ ] `net/NetworkSession.java` → `ucc-api/.../api/net/NetworkSession.java`
  - 移除对 `Piece.Type` 的未使用 import
  - 更新 `UpdateInfo` import → common
- [ ] `net/NetModeController.java` → `ucc-api/.../api/net/NetModeController.java`
